## 핵심 플레이 흐름

```
게임 시작 → 미니게임 랜덤 선택 → 지시문 표시 (1초) →
미니게임 플레이 (10-15초) → 성공/실패 판정 →
다음 미니게임 or 게임 오버
```

- **30개 미니게임**을 각 1회씩 플레이
- **매번 다른 순서**로 랜덤 등장
- **5판마다 난이도 상승** (Lv.1 → Lv.6)

## 점수 시스템

### 미니게임별 점수
- **성공 횟수 기반**: 성공 × 10점
- **정확도 기반**: 정확도 × 100점
- **시간 기반**: 남은 시간 보너스

### 최종 점수
```
총점 = Σ(미니게임 점수) + 난이도 보너스 + 하드모드 보너스
```

## 개발 주의사항

### ⚠️ 이벤트 처리 (중요)

> **웹뷰 환경**에서 동작하므로 이벤트 처리에 각별히 주의해야 합니다.

| 요구사항 | 설명 |
|----------|------|
| **터치 + 클릭 동시 지원** | 모든 인터랙션은 `@touchstart` + `@click` 모두 구현 |
| **터치 기본동작 방지** | `@touchstart.prevent`로 스크롤/줌 방지 |
| **좌표 변환** | `getCanvasCoordinates()`로 캔버스 좌표 정확히 계산 |
| **이벤트 중복 방지** | 터치 디바이스에서 click 중복 발생 주의 |

```vue
<!-- 올바른 이벤트 바인딩 예시 -->
<canvas
  @touchstart.prevent="handleTouch"
  @click="handleClick"
></canvas>
```

## UI/UX 가이드

### 메인 컬러
| 용도 | 컬러 | 코드 |
|------|------|------|
| Primary | 노랑 | `#FFD700` |
| Secondary | 주황 | `#FF9800` |
| Accent | 보라 | `#5E35B1` |
| Success | 초록 | `#4CAF50` |
| Error | 빨강 | `#F44336` |

### 디자인 원칙
- **이미지 없이** Canvas + CSS + 이모지로 구현
- **둥근 모서리** (12-20px)
- **단순한 그라데이션** (5-10% 명도 차이만)
- **명확한 피드백** (진동, 파티클, 점수 팝업)

### 피드백 시스템
| 이벤트 | 시각 | 햅틱 |
|--------|------|------|
| 성공 | 파티클 + 점수 팝업 | 짧은 진동 (25ms) |
| 콤보 | 큰 팝업 + 흔들림 | 연속 진동 |
| 실패 | 빨간 플래시 | 강한 진동 (100ms) |
| 게임 오버 | 화면 흔들림 | 긴 진동 패턴 |
